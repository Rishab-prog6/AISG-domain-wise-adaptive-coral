核心问题

从 CORAL 到 ANCHOR：不换模型，换对齐方式

① 对齐项是否在正确尺度？

协方差是 d×d（d=512），直接用 Frobenius 范数让 CORAL 项达 10⁴~10⁶、淹没交叉熵(~2)。→ 标准缩放 /(4d²) 压回同量级。

② 应该对齐到谁？

CORAL 对齐到平均协方差 C̄，会被抽象域拉偏。→ ANCHOR 改为对齐到最低损失源域的 detach(C_anchor)。

两者共享的目标函数骨架（改动只在对齐项的 TARGET）

loss = mean_e CE(f(x_e), y_e)  +  λ · mean_e ‖ C_e − TARGET ‖²_F / (4d²)
       CORAL:  TARGET = C̄（所有源域协方差平均）
       ANCHOR: TARGET = detach(C_anchor)（当前最低损失的源域）
原版 CORAL · 机制与代价

CORAL 帮难域，却伤自然域

收益：抬高强风格偏移域

让源域二阶统计量接近，分类器更难依赖域独有的纹理/画风。
Sketch：ERM 73.50 → CORAL 79.49（+5.99）。

代价：拉偏自然域

平均目标 C̄ 被 Sketch/Cartoon 拉向抽象风格，把 Art 往抽象方向拉。
Art：ERM 81.17 → CORAL 78.24（−2.93）。

ERM → CORAL 的"削峰填谷"（PACS，百分点）

CORAL 不是免费午餐：抬高抽象域（Sketch +5.99）的同时压低自然域（Art −2.93, Photo −3.89）。这正是 ANCHOR 想改善的取舍 →
最终方法 · DWA_CORAL_ANCHOR_ALIGNONLY

ANCHOR：对齐到锚点，而非平均值

① 分类不加权

cls = mean_e L_e，避免因难域 loss 高扭曲监督信号，保住简单域分类边界。

② 锚点 detach

C_anchor 只作目标、不被对齐梯度拉动，保留最稳源域的判别流形。

③ 非锚域自适应

对齐权重 = 半均匀 + 半 loss-softmax，困难域获更多对齐压力。

核心公式

anchor  = argmin_e detach(L_e)
coral_e = ‖ C_e − detach(C_anchor) ‖²_F / (4d²)
w_other = (1−γ)·均匀 + γ·softmax(τ·detach(L_other))
cls     = mean_e L_e        # λ=0.3, τ=1.0, γ=0.5

ANCHOR · 直觉：为什么 Art↑ 而 Cartoon↓

它改变的是对齐的方向

Art 为什么变好 ↑

Art 更接近自然图像流形。CORAL 平均目标被抽象域拉偏；ANCHOR 用低损失源域作 detached 参照，减少拉偏，Art 78.24 → 79.71。

Cartoon 为什么变差 ↓

Cartoon 更抽象。锚点偏自然图像，Cartoon 被拉向它后自身风格特征被压缩，76.28 → 74.79。

Photo、Sketch 与 CORAL 基本不变——ANCHOR 把代价从 Art 转移到 Cartoon。

④ 问题 / 挑战 / 解决方案 · 结论

踩过的坑与 解决方案

挑战 / 问题现象解决方案
CORAL 项尺度爆炸准确率塌到 ~20%（近随机）除以 4d² 标准归一化 → 80.08%
加权分类损失伤简单域v2≈CORAL，Cartoon/Photo 掉改为只加权对齐（ALIGNONLY）
平均目标被抽象域带偏Art 被伤锚定最易域单向对齐（ANCHOR）
退火想再拉回 Art反而更差（不变性发散）git revert 回退，留作负结果
单种子噪声 ±1%不能宣称显著超越结论克制 + 建议 ≥3 种子复现
不能改官方代码保证可比性猴子补丁架构（dwa_train_v2）

结论

不是"全面更强"，而是改变了收益分布：ANCHOR 把 CORAL 牺牲 Art 的代价转移到 Cartoon，Avg 打平、Art 追回 +1.47，并在 VLCS 复现。
